# Zesu documentation

This repository contains the source for [Zesu](https://docs.zesu.consensys.io), the Consensys
stateless ZK execution client documentation site. It is built with Docusaurus.

## Documentation structure

| Folder              | Path                  | Description                                              |
|---------------------|-----------------------|----------------------------------------------------------|
| Concepts            | `docs/concepts/`      | Explanation of Zesu's design and how it works.           |
| Get started         | `docs/get-started/`   | Onboarding: system requirements, installation, quickstart. |
| How-to              | `docs/how-to/`        | Goal-oriented guides for configuring and operating Zesu. |
| Reference           | `docs/reference/`     | CLI options and other reference material.                |
| Troubleshooting     | `docs/troubleshooting.md` | Common problems and fixes.                           |

The sidebar is **manually defined** in `sidebars.js`. New pages must be added to `sidebars.js`
before they appear in the navigation.

## Editorial standards

Follow these guides when writing or editing documentation:

- [Consensys documentation style guide](https://docs-template.consensys.net/contribute/style-guide)
- [Consensys Markdown formatting guide](https://docs-template.consensys.net/contribute/format-markdown)
- [Microsoft Writing Style Guide](https://learn.microsoft.com/en-us/style-guide/welcome/)
- [Diataxis framework](https://diataxis.fr/) for content structure

The rules under `.cursor/rules/` cover the most actionable parts of these references.

## Critical rules

1. **Do not invent product behavior.** Never state as fact any CLI option, configuration setting,
   API parameter, return value, or behavior that you have not verified against the Zesu codebase
   or published reference. If you are uncertain, flag it for review.
2. **Update `sidebars.js` for new pages.** Sidebars are not autogenerated. Add new pages to
   `sidebars.js` in the correct category and position before requesting review.
3. **Use the canonical terminology.** See `.cursor/rules/terminology.mdc` for the required spelling
   and casing of product and industry terms.
4. **Verify before publishing.** Preview changes locally with `npm start` before requesting
   review. Docusaurus is configured with `onBrokenLinks: 'throw'`, so broken internal links fail
   the build.
5. **Pass the CI linter.** PRs are checked by the
   [Consensys docs-lint-all](https://github.com/Consensys/github-actions) action that enforces
   Microsoft style and Consensys terminology. Fix warnings before requesting review.

## AI guidance

Detailed editorial, formatting, and product-specific rules are in `.cursor/rules/`. These rules
apply automatically when you edit files matching their glob patterns.

Agent skills live under `.cursor/skills/<skill-name>/SKILL.md`. This repo includes **author-page**
(scaffold and draft new pages) and **style-review** (editorial compliance before PR). Cursor loads
these when the task matches each skill's description.
